◈ Screen 07

Recovery & Resumption Flow

The end-to-end journey from interruption to confident resumption. Each node represents a system decision point or user-facing screen. The flow shows how Resumate intelligently routes users through the minimal path back to productive work.

⚡
Interruption Occurs
Tab closed, offline, idle 5min, device switch, or voluntary exit
→
⚙
Auto-Detection
SessionService classifies interruption type and triggers immediate localStorage save
→
💾
State Preserved
Section, field, scroll, drafts, AI context, device ID, action log -- all persisted in <100ms
↓
User returns (minutes, hours, or days later)
👁
Return Detection
GlobalSessionService checks lastActiveAt timestamp, compares deviceId, queries backend for newer state
→
◆
Decision Engine
• Away < 2min: Silent (no UI)
• Away > 2min: Welcome Back Guide
• On /editor: + Recovery Screen
• Device mismatch: + Sync Banner
• Pending apps: + Nudge Widget
→
🎨
Tailored Re-entry UI
Exactly the right overlay, banner, or widget -- never more than needed, never less
↓
User takes action (one click)
▶
One-Click Resume
Scroll to section, focus last field, restore draft text, show "Restored" tooltip
→
✎
Productive in <10 seconds
User is typing in the exact field they left, with full context visible in sidebar. Zero cognitive overhead.
→
↺
Continuous Protection
10s autosave resumes, idle timer resets, action log appends. Ready for the next interruption.
